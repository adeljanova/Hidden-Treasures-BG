Интерактивна визуализация на UML архитектурата на софтуерния проект

1. UML Класова Диаграма (Class Diagram)

Показва ООП структурата, наследяването от базовия клас PlaceOfInterest към конкретните типове ресурси, както и композицията в класа Village.

classDiagram
    class PlaceOfInterest {
        +String name
        +String description
        +String coordinates
        +displayInfo()
        +getName()
    }

    class CulturalResource {
        +String century
        +String typeOfCulture
        +displayInfo()
    }

    class NaturalResource {
        +String naturalFormation
        +Boolean isProtectedArea
        +displayInfo()
    }

    class TouristAttraction {
        +Double entryFee
        +Integer rating
        +displayInfo()
    }

    class Village {
        +String name
        +String region
        +addPlace()
        +displayAllPlaces()
        +getName()
    }

    class Application {
        +run()
        +showMenu()
        +seedData()
    }

    PlaceOfInterest <|-- CulturalResource
    PlaceOfInterest <|-- NaturalResource
    PlaceOfInterest <|-- TouristAttraction
    Village o-- PlaceOfInterest
    Application *-- Village

2. UML Диаграма на Случаите на Употреба (Use Case Diagram)

Описва основните действия, които потребителите могат да извършват в рамките на конзолното приложение.

graph LR
    classDef actor fill:#38bdf8,stroke:#0284c7,stroke-width:2px,color:#fff;
    classDef case fill:#1e293b,stroke:#334155,stroke-width:2px,color:#f8fafc;

    User((Потребител)):::actor
    
    UC1(Разглеждане на населени места):::case
    UC2(Търсене на обект по категория):::case
    UC3(Преглед на детайли за обект):::case
    UC4(Добавяне на ново населено място):::case
    UC5(Добавяне на нов обект):::case
    
    User --> UC1
    User --> UC2
    User --> UC3
    User --> UC4
    User --> UC5
